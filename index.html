Routing by Query Parameter in Lithium

The following routing pattern can be used to map SOAP like web service requests where you have one API endpoint and method calls are passed via a query parameter to RESTful URLs. Example: /Products?method=getAll => /Products/getAll

Router::connect(
  '/{:controller}', 
  array('controller' => '{:controller}'),
  function($request) {
    if (isset($request->query['action'])) {
        $request->params['action'] = $request->query['action']
    }
  }
);

Lithium Quick Tip for Queries With Join

When you are querying Models with relationships and you want to include the joined Models in your result list don’t forget to use instances in your conditions. Example:

1
2
3

Users::first(1, array(
  'with' => array('Posts')
));

This should return an error message about ambiguous ids. To fix the error you’ll need to do the following:

1
2
3
4
5
6
7
8

Users::first(array(
  'conditions' => array(
    'Users.id' => 1 // Note the instance name
  ),
  'with' => array(
    'Posts'
  )
));

Also watch out for case-sensitivity; code running on Windows might fail on Linux due to inconsistent namings, this applies to your database table names as well. Stick to the conventions and you’ll be fine.
